## 개요📜

>controller, service, repository 에 해당하는 기능들을 계층별로 분리합니다.    
>각 layer 는 인터페이스를 통해 느슨한 의존성을 가질 수 있도록 합니다.    
>느슨한 의존을 통해 구현 방식 변경 등으로 인한 수정이 최소화 될 수 있도록 합니다.

***   

## 배경🖼️

>현재 controller 는 외부에 노출되는 인터페이스를 정의함과 동시에 로직을 포함하고 있습니다.  
>그로 인해 서로 다른 다양한 업무가 하나의 코드에서 충돌할 수 있는 위험한 상황이 예상됩니다.
***

## 목표📌

1. controller 에서 로직 제거
2. controller, service, repository 의 계층을 을 분리
3. 각 계층을 잇는 인터페이스 정의
***

## 목표가 아닌 것✂️

- API 스펙 변경
- 기능 수정
